# Systems Notes

## Purpose

This folder is for normalized system-level notes, not rough idea dumps.

Each system note should answer:

- what the system does
- where the system starts and stops
- what utilities it depends on
- what the main control objectives are
- what the main permissives, interlocks, and trips are
- what instruments and devices are typical
- what standards or code families shape the design
- which manuals and public sources support the note

## Suggested note pattern

- Purpose and boundary
- Major equipment blocks
- Control objectives
- Main measurements and devices
- Safety and shutdown behavior
- Standards anchors
- Source links
- Promotion target
