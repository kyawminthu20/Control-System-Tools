## Topic focus

This segment teaches an exam-prep workflow for locating NEC answers quickly without reading large sections blindly.

## Main concepts captured

### 1. Start by classifying the question

- The transcript separates questions into two broad types:
  - lookup questions
  - calculation questions
- The lookup process is treated as a repeatable method, not a guess.

### 2. Reduce the question to the real subject

- The speaker's "keyword and index" method starts by asking what the scenario is actually about.
- The first keyword should come from the real installation problem, not from random words in the question stem.

### 3. Use the NEC index as a routing tool

- Go to the back-of-book index.
- Use bold headings first to get to the correct subject area.
- Only read the smaller nested entries after the main subject is confirmed.

### 4. Refine with a second keyword

- Once the first subject heading is found, use the next key concept from the question to narrow the route.
- The method is iterative rather than one-step.

### 5. Use answer choices to judge whether a section is promising

- The transcript recommends scanning headings and nearby content before reading every paragraph in full.
- Multiple-choice answer values can help indicate whether the current section is worth deeper reading.

### 6. Read the full paragraph once the route is confirmed

- The note stresses not stopping at the first phrase that looks correct.
- Full paragraph reading is still necessary because later wording may limit or modify the apparent answer.

## Working takeaway

This segment is really a code-navigation method:

- identify the real subject
- route through the index
- confirm with headings
- then slow down and read the actual NEC text carefully
