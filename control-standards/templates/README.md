# Templates
**AI_READ_ACCESS: ALLOWED (for template reference)**

Practical starter templates for creating new repository content that matches the current project structure and metadata conventions.

Use these templates as starting points only. Adjust metadata, scope, and section headings to fit the actual content being created.

## Contents

### `md_headers/`

Current metadata/header starters:

- `rag_approved_header.md`
  For authoritative AI-readable content under `control-standards/rag/`
- `draft_only_header.md`
  For `work/design/` and other non-authoritative WIP notes
- `archived_header.md`
  For archived or superseded material

### `checklists/`

- `checklist_template.md`
  Starter for commissioning or review checklists

### `design_guides/`

- `design_guide_template.md`
  Starter for applied engineering guides and workflows

### `reports/`

- `report_template.md`
  Starter for structured reports or generated outputs

### `work_notes/`

- `work_note_template.md`
  Starter for transcript-derived notes, source notes, and normalized WIP material

## Usage

1. Copy the appropriate template into the target area:
   - `control-standards/rag/` for authoritative promoted content
   - `control-standards/work/design/` for source notes and WIP material
   - `control-standards/archive/` for archived references
2. Rename the file to match the actual content.
3. Replace placeholder metadata and headings.
4. Add standards links only when there is a real standards anchor.
5. Keep heuristics, OEM instructions, and standards requirements clearly separated.

## Important rule

These templates reflect the **current** repository conventions.

Do not fall back to the older simplified header style such as:

- `**Status:** Authoritative`
- `CONTENT_CLASS: PROMOTED`
- `SOURCE:`
- `PROMOTION_TARGET:`

unless the repository is explicitly migrated to those conventions everywhere.
