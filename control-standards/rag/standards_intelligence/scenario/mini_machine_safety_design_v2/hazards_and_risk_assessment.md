## Risk reduction hierarchy

The local corpus supports the following working hierarchy:

1. Reduce risk by design so equipment failure does not create a hazardous condition. [LOCAL: international/machinery/iec_60204_1/IEC60204_1_2018__Clause04__general_requirements.md#0. Safety Philosophy]
2. Apply protective measures in control functions, isolation devices, enclosures, barriers, and bonding systems. [LOCAL: us/nfpa79/NFPA79_2024__Ch05__disconnecting_means.md#1. ## Concise risk assessment template

Use this template for each identified hazard:

| Field                    | Entry guidance                                             |
| ------------------------ | ---------------------------------------------------------- |
| Hazard ID                | Unique tag such as `HZ-HYD-01`                             |
| Hazard description       | Plain-language hazard statement                            |
| Operating mode           | Auto, manual, setup, maintenance, cleaning                 |
| Initiating event         | Fault, operator action, maintenance action, loss of power  |
| Exposure zone            | Operator station, service side, guarded zone, chemical bay |
| Consequence              | Injury, release, damage, fire, contamination               |
| Existing safeguards      | Device or design feature with function                     |
| Additional action        | Design change, label, procedure, validation                |
| Required safety function | Link to `SF-xx` where applicable                           |
| Verification method      | Inspect, test, calculate, review record                    |
| Standard trace           | Local citation or `TO VERIFY`                              |
